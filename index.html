Step 1 요구사항 정의

PM 업무 5가지를 AI가 잘하는 능력 단위로 분석·재분류.
"사람 기준 업무 칸막이"에서 "AI 능력 분류"로 관점 전환.

Step 2 Agent별 인터뷰

CLAUDE_1201~1205 — 5개 Agent 각각 7라운드 질문형 인터뷰.
실제 문서를 참조하며 진행. 라운드당 3개 이하 질문.

Step 3 워크시트 통합

5개 개별 워크시트를 통합 인터뷰로 재구성.
Agent 간 데이터 흐름, 긴급도 순서, 빈칸 항목 도출.

Step 4 프로젝트 설계서

Claude Code 공식 Agent-Skill 가이드 참고.
폴더 구조·Agent 정의·Skill 할당·model·tools 일괄 설계.

Step 5 구현 & 병목 발견

Agent 5개 + Skill 12개를 만들었으나 산출물 0건.
"설계 완료"와 "실행 검증"은 완전히 다른 문제.

Step 6 보완 인터뷰 & 최적화

중급 자료(automation-pipeline-design-eduv) 활용 재인터뷰.
Guardian orchestrator Agent 추가. 실행 검증 완료.

Agent별 인터뷰: doc-manager 예시

doc-manager 시뮬레이션 Agentic RAG

정부과제 지원사업 (사업신청서, 사업계획서 등) hwp 필수문서를 AI 관점에서 작업한다고 생각하고 인터뷰 해줘
Round 1 — 현재 고통
문서 관리에서 가장 큰 어려움은 무엇인가요?
4개 팀에서 hwp·pdf·md·docx 다양한 형식으로 문서가 들어오는데, 어디에 뭐가 있는지 파악이 안 됩니다.
Round 2 — 입력 · 문서 유형
어떤 종류의 문서를 다루나요? 형식과 출처를 알려주세요.
정부과제: 공고문·사업계획서·신청서류 (.hwp) / 개발: 외주 산출물·기술문서 (.md/.pdf) — 2개 계열, 12종 이상입니다.
Round 3 — 처리 흐름 · 예외
hwp 파일은 AI가 어떻게 처리하면 좋겠나요?
hwp는 직접 편집 불가 → md로 변환 후 작업, 제출용만 hwp로 재변환. 원본은 보존해야 합니다.
Round 4 — 출력 · 핵심 기능
이 Agent가 반드시 해야 할 일 3가지는 무엇인가요?
① hwp ↔ md 변환 ② 필수서류 체크리스트 ③ 문서 인덱스 관리
Round 5 — 저장 · 전달
이 Agent가 데이터를 주고받는 상대 Agent는?
커뮤니티에서 컨설팅 피드백 수신, 리서치에 기존 참고자료 전달 — 최다 수신 허브 역할입니다.
질문 1회 1개 · 총 7라운드 · 실제 문서(.hwp) 참조하며 진행

설계서가 자동 결정한 Agent · Skill frontmatter

frontmatter란? — .md 파일 맨 위 --- 블록에 적는 메타데이터. Claude Code가 이 정보를 읽고 Agent의 모델·도구·설명을 자동으로 인식한다.

frontmatter가 없다면?
"doc-manager Agent 만들어줘. 모델은 sonnet, hwp 파싱용 Bash 필요해"
→ Claude Code가 frontmatter를 자동 생성

Agent frontmatter 실제 예시

--- doc-manager.md ---
name: doc-manager
description: "[21.01] 문서관리 Agent"
model: sonnet        # ← 자동 결정
tools:               # ← 자동 결정
  - Read
  - Write
  - Edit
  - Bash              # hwp_parser.py
  - Glob              # 문서 탐색
  - Grep              # 내용 검색

Skill frontmatter 실제 예시

--- convert-hwp/SKILL.md ---
name: convert-hwp
description: "[31.01] hwp ↔ md 변환"
agent: doc-manager   # ← 소속 Agent
aliases:
  - hwp 변환해줘        # ← 자연어 호출
  - md로 변환해줘
tools:
  - Read, Write, Edit, Bash, Glob
Skill은 Agent에 소속, aliases로 자연어 호출 | Skill 번호(3x) = Agent 번호(2x) 대응

무엇이 문제였나?

  • 설계는 완벽했지만 실행 검증이 없었다
    Agent 5개·Skill 12개를 만들었으나 실제 산출물은 0건
  • doc-manager에서 병목
    hwp 파싱 실패, 파일 배치 규칙 위반, change_log 누락 — 후속 Agent가 진행 불가
  • 규칙 준수를 검증하는 주체가 없었다
    PM이 매번 직접 확인해야 함 → 자동화의 의미 상실
  • Agent 간 직접 호출 시 예측 불가
    누가 누구를 호출했는지 추적 불가, 충돌 발생

→ 중급 자료(automation-pipeline-design-eduv)로
　시뮬레이션 테스트 + 재인터뷰 진행
